# Agentic 翻页双语站设计

## 目标

将当前长页演示站重构为一个面向现场投屏主讲的全屏翻页网站。

新网站的核心目标：

- 只保留翻页体验，不再保留长滚动页作为正式入口
- 每个部分独立成一页，强调演示节奏和章节感
- 保持当前项目已形成的视觉质量、科技感和信息结构
- 引入中英双语，但不让双语排版破坏当前已经成熟的版式
- 旧长页版本需要完整备份，确保可回滚、可参考、可复用

## 非目标

- 不做一套“长页版 + 翻页版”并存的正式对外体验
- 不做逐句中英双栏对照
- 不为了双语重做全部视觉系统
- 不引入复杂的 3D 翻转或高风险动效

## 最终方案

采用 **左右翻页的全屏章节站**。

每个部分对应一个独立章节，用户通过以下方式切换章节：

- 键盘左右方向键
- 鼠标滚轮横向捕捉
- 屏幕侧边章节导航
- 页内显式上一页 / 下一页控制

章节切换动效采用轻量科技感方案：

- 主体滑动切换
- 背景光晕微位移
- 章节内容分层淡入

不采用重 3D、卡片翻面、长时间转场等会抢内容注意力的方案。

## 双语规则

双语采用 **中文主叙事，英文辅助表达**。

固定规则如下：

- 中文承担主标题、主说明、主图表文案
- 英文承担副标题、眉题、标签、关键动作词和补充短句
- 同一页内允许中英共存，但不做左右对称双栏翻译
- 英文信息必须服务理解和国际化表达，不能压缩中文主内容的阅读空间

这条规则是为了避免后续继续调格式。实现时应优先复用现有中文版式，再把英文嵌入到不破坏结构的位置。

## 章节结构

新网站按以下 7 个章节组织：

1. Opening
2. Current State
3. Future State
4. Product Structure
5. Demo
6. Scenarios
7. Toolkit

其中：

- `Opening`：用一句中文主标题 + 英文副标题建立主题
- `Current State`：复用现有“现状与痛点”内容
- `Future State`：复用现有 “Agentic 升级”内容
- `Product Structure`：保留发现 / 理解 / 解决链路
- `Demo`：作为独立章节，承担过渡和展示作用
- `Scenarios`：4 类热门场景保留在 **同一页**，不拆页
- `Toolkit`：作为收束页，标题更新为 `Toolkit × Agentic Hub`

## 场景页设计要求

`Scenarios` 章节必须保持一页容纳四类场景，不拆分为两页。

为避免该页因为双语而变挤，规则如下：

- 四类场景继续保持卡片式结构
- 中文描述为主
- 英文只出现在标题、副标签、关键动作词上
- 不在每张卡片中额外加入完整英文段落

这样可以保证：

- 一页完整展示 4 类场景
- 投屏时信息密度仍可控
- 不需要重新为四张卡做复杂的中英重排

## 信息架构与导航

全站采用统一的章节导航组件，包含：

- 当前章节序号，例如 `01 / 07`
- 右侧竖向章节进度条
- 当前章节名称
- 上一页 / 下一页控制

导航应该始终稳定存在，但视觉上低于内容主体，避免打断演示。

## 视觉规则

视觉沿用当前项目的深色科技感语言：

- 深色背景
- 玻璃感半透明面板
- 柔和光晕
- 层级化边框
- 大字号标题 + 框图式内容承载

针对翻页站新增的视觉规则：

- 每一页必须有清晰的“章节开头感”
- 大标题区、编号区、内容区之间有明确层级
- 动效服务“翻页”和“章节切换”，不服务装饰
- 主要框格允许继续使用 0.3 边框基线，并按层级使用白色或轻微色相边框区分

## 技术实现边界

新网站直接替换当前正式入口，不继续保留长页作为正式首页。

实现边界如下：

- 默认入口改为翻页站
- 当前长页相关页面与关键组件在改造前完整备份
- 新站章节化组织内容，但尽量复用已有文案、资产和图表数据
- 不直接硬切现有整页 DOM，而是以“章节”为单位重组内容

建议实现形态：

- 一个新的翻页容器页负责全局切页逻辑
- 每个章节一个独立 scene / section 组件
- 中英文案进入统一数据层，而不是零散写死在 JSX 中

## 备份策略

旧长页必须保留可回滚版本。

要求：

- 在项目内建立明确命名的备份目录
- 备份当前首页、相关章节组件和必要测试文件
- 备份目录命名必须带日期和用途
- 新站切换完成后，旧版不作为正式入口，但必须可恢复

## 验证方式

实现完成后至少验证以下内容：

- 默认入口为翻页站
- 左右翻页逻辑稳定
- 7 个章节都能访问
- `Scenarios` 保持单页四卡布局
- 中英文案符合“中文主叙事，英文辅助”规则
- 投屏下标题和关键框图可读性稳定
- 旧长页备份存在且路径明确

工程验证至少包括：

- 路由 / 渲染级测试
- 构建验证
- 关键翻页交互的手工验收

## 风险与约束

主要风险：

- 直接把中英并列到同层，会破坏当前版式
- 一页四类场景如果英文过多，会导致投屏密度过高
- 动效如果太重，会削弱演示内容本身

对应约束：

- 不做中英双栏对照
- 不拆分第 6 部分
- 不做高风险重动效
- 不保留长页作为正式入口

## 推荐结论

最终推荐方案是：

- **交互**：左右翻页全屏章节站
- **语言**：中文主叙事，英文辅助表达
- **结构**：7 个章节，其中 `Scenarios` 固定为单页四卡
- **入口**：新翻页站替代长页作为正式入口
- **备份**：旧长页完整存档，可随时回滚
