**产品**：TikTok Ads Manager (TTAM)
**来源**：TikTok for Business Help Center + 产品实际界面浏览
**整理日期**：2026-06-04
**版本**：v1.0

---

## 一、四种竞价策略概述

| 策略 | 中文名 | 工作原理 | 适合场景 | 核心优势 |
|------|--------|---------|---------|---------|
| **Lowest Cost (Cost Cap / 最低成本)** | 自动出价 | 系统在不设价格上限的前提下寻找最低价转化 | 新账户冷启动 / 想要最大转化量 / 对 CPA 没有严格上限 | 最大化转化量，操作最简单 |
| **Bid Cap (竞价上限)** | 手动出价 | 你设置最高可接受的单次点击/千次展示价，系统严格遵守 | 对单次互动成本有明确预算上限 / 需要严格控制投放节奏 | 成本可预测、可控 |
| **Cost Cap (成本上限)** | 智能出价 | 你设置目标平均 CPA，系统在保持 CPA 低于目标的前提下最大化转化量 | 数据量充足 + 有明确 CPA 目标的转化类广告 | 稳定 CPA + 效率兼顾 |
| **Smart+ (自动化系列)** | AI 全自动化 | 系统决定出价、受众、版位、创意（可部分覆盖设置） | 新广告主 / 规模化投放 / 没有时间做精细化运营 | 省时省力、转化效率高 |

> **2024-2026 年重要变化**：Cost Cap 经历过一次规则升级——新 Cost Cap 不再是"目标 CPA"，而是"不超过这个平均成本"，系统会主动压低竞价以获取更多低成本转化；Smart+ 新增"升级版体验"，允许在自动化框架内部分覆盖设置。

## 二、Lowest Cost（最低成本）— 最常用的默认策略

### 2.1 工作原理

这是 TikTok 最推荐的冷启动策略。**算法不给自己设任何竞价上限**，只关心一个目标：在你的预算内拿到最多转化。

```
预算 = $100/天
目标 CPA = 不指定（系统自动找最低价）

第 1 天：系统激进探索，CPA 可能偏高 → 这是 Learning Phase
第 2-3 天：算法学到用户特征，逐渐收敛到稳定 CPA
第 4+ 天：稳定期，CPA 保持在市场合理水平
```

### 2.2 优势与劣势

| ✅ 优势 | ❌ 劣势 |
|--------|--------|
| 系统学习效率最高，走出学习期最快 | 初期 CPA 不稳定，需要预算容忍度 |
| 可拿到最大转化量 | CPA 可能比你预期的高（尤其是库存紧张时） |
| 操作简单，适合 A/B 测试基线 | 不适合对 CPA 有严格 KPI 要求的场景 |
| 与 Smart+ 搭配的最佳基础策略 | — |

### 2.3 推荐使用场景

- 新 Pixel / 新账户冷启动
- 短期促销活动（走量优先）
- 创意 A/B 测试（需要系统公平分配预算）
- 预算相对宽裕、更关注转化量而非 CPA

### 2.4 最低预算要求

| 层级 | 最低日预算 |
|------|-----------|
| **Ad Group（广告组）** | $20 |
| **Campaign（推广系列 + CBO）** | $50 |

> **经验法则**：日预算 ≥ 目标 CPA × 20（保证一天至少 20 次转化供算法学习）

## 三、Bid Cap（竞价上限）— 成本高度可控

### 3.1 工作原理

你手动设置每次点击 / 千次展示 / 单次转化的最高可接受价格。系统会**严格遵守这个上限**，不会在任何一次竞价中超出。

```
Bid Cap = $2 CPC

如果一次预估 CPC = $2.5 → 系统放弃这次竞价
如果一次预估 CPC = $1.8 → 系统参与竞价
如果预估 CPC 长期低于 $2 → 转化量可能比 Lowest Cost 少但成本稳定
```

### 3.2 优势与劣势

| ✅ 优势 | ❌ 劣势 |
|--------|--------|
| 成本完全可控，CPA 可预测 | 学习期更长，系统优化空间受限 |
| 适合低毛利行业（CPA 必须 < 某个值） | 转化量通常比 Lowest Cost 少 15-30% |
| 配合创意测试可做"质量优先"策略 | 设置过低会导致投放消耗不足 |

### 3.3 推荐使用场景

- 对 CPA 有严格 KPI 限制（例如毛利低、需要保证 ROI）
- 已验证创意/受众，进入稳定放量阶段（需精细控本）
- 特定行业（金融、教育、B2B SaaS，线索成本明确）

### 3.4 如何设置合理的 Bid Cap

1. 先用 Lowest Cost 跑 7 天，看实际平均 CPA
2. 将目标 CPA 设置为「平均 CPA × 0.8」（压低 20%）作为初始 Bid Cap
3. 观察 3 天：
   - 如果消耗不足 → 上调 10-20%
   - 如果 CPA 远高于目标 → 下调或切回 Lowest Cost

## 四、Cost Cap（成本上限 / oCPA）— 兼顾效率与可控

### 4.1 工作原理

TikTok 的智能出价策略。你设置一个**目标平均 CPA**，系统在不突破该目标的前提下最大化转化量。与 Bid Cap 不同的是，Cost Cap 允许**单次转化高于目标**，只要整体平均 CPA 不超过目标即可。

```
目标 CPA = $15

第 1 次转化：$25（高于目标，但系统允许）
第 2 次转化：$12
第 3 次转化：$8
第 4 次转化：$14
平均 CPA = $14.75 ✅（低于目标）
```

> **2024 年升级要点**：新版 Cost Cap 更偏向"先拿低价转化再拓展高价"，与旧版（强制贴近目标 CPA）不同。现在系统会主动压低竞价，等低价转化被消耗完才抬高。

### 4.2 前提条件

- **数据要求**：Pixel 过去 7 天累计 ≥ 50 次转化（或 15 次/天）
- **事件选择**：建议优化漏斗中上层事件（View Content / Add to Cart），而非直接优化 Complete Payment（样本过少）
- **受众规模**：建议受众 ≥ 100 万（太小的受众系统学不到规律）

### 4.3 优势与劣势

| ✅ 优势 | ❌ 劣势 |
|--------|--------|
| 稳定 CPA + 相对高转化量 | 需要足够数据（新 Pixel 不建议） |
| 比 Lowest Cost 更可控，比 Bid Cap 更灵活 | 目标 CPA 设置过高会浪费预算 |
| 适合长期稳定投放 | 目标 CPA 设置过低会导致学习失败 |

### 4.4 推荐使用场景

- 已有 ≥ 50 次转化的成熟账户
- 需要稳定 CPA、同时关注转化量增长
- 电商购买 / App 安装 / 表单提交等高频转化事件

### 4.5 设定策略

| 阶段 | 建议目标 CPA |
|------|-------------|
| **稳定期（≥ 200 次转化/周）** | 目标 CPA = 历史 7 天平均 CPA × 1.0（保持原水平） |
| **扩张期（想放量）** | 目标 CPA = 历史 7 天平均 CPA × 1.1-1.2（提高 10-20%） |
| **收缩期（想降本）** | 目标 CPA = 历史 7 天平均 CPA × 0.8-0.9（降低 10-20%，需谨慎） |

## 五、优化事件（Optimization Events）完整列表

在 Ad Group 设置中，你选择的优化事件决定了系统按什么指标优化投放。

### 5.1 转化类事件（Conversion）

| 事件 | 中文 | 说明 |
|-----|------|------|
| **View Content** | 浏览内容 | 落地页浏览 / 商品详情页浏览 —— 最容易触发的事件 |
| **Add to Cart** | 加入购物车 | 漏斗中段，比 View Content 更能反映购买意向 |
| **Add Payment Info** | 添加支付信息 | 高意向信号，接近转化 |
| **Initiate Checkout** | 开始结算 | 强购买意向信号 |
| **Complete Payment / Purchase** | 完成支付 | 核心转化事件 —— 最有价值但样本最少 |
| **Complete Registration** | 完成注册 | SaaS / 会员 / App 注册 |
| **Search** | 搜索 | 用户在网站内搜索 |
| **Contact** | 联系 | 用户发起联系（表单提交 / WhatsApp / 电话） |

### 5.2 互动类事件

| 事件 | 中文 | 说明 |
|-----|------|------|
| **Click** | 点击 | 按点击数优化（Traffic 目标） |
| **Impression** | 曝光 | 按最大曝光优化（Reach 目标） |
| **2s View / 6s View** | 2 秒 / 6 秒观看 | 视频广告的观看深度（Video Views 目标） |
| **Engaged View** | 参与观看 | 6 秒观看 + 后续互动（新版 Video Views 目标） |

### 5.3 应用类事件

| 事件 | 中文 | 说明 |
|-----|------|------|
| **Install** | 安装 | App 下载并首次打开 |
| **App Purchase** | 应用内购买 | App 内完成的交易 |
| **App Register** | 应用内注册 | App 内完成注册 |
| **Level Achieve / Tutorial Complete** | 游戏成就事件 | 游戏类 App 常用 |

> **优化事件选择原则**：选择一个**发生频率足够高**（≥ 20-50 次/周）且**离购买越近越好**的事件。新 Pixel 通常从 View Content 开始，积累数据后切到 Add to Cart，最终优化 Complete Payment。

## 六、预算策略：日预算 vs 总预算

### 6.1 日预算 (Daily Budget)

- 系统每天花掉的金额上限
- 实际单日花费可能在 ±20% 内浮动（系统会根据流量自动调整）
- 适合长期持续投放
- Ad Group 最低 $20/天；CBO Campaign 最低 $50/天

### 6.2 总预算 / 生命周期预算 (Total Budget / Lifetime Budget)

- 从开始到结束的总预算上限
- 系统在投放周期内自动分配，可能前期消耗快、后期慢
- 适合有限预算的短期活动（如节日促销、新品发布）
- 需设置开始和结束时间

### 6.3 推荐选择

| 场景 | 推荐预算类型 |
|-----|------------|
| 品牌日常投放（持续 ≥ 2 周） | 日预算 |
| 短期促销（1-2 周后结束） | 总预算 + 排期 |
| 测试新创意（快速验证） | 日预算 + 小规模（$50-100/天） |
| Smart+ 自动化系列 | 日预算（Smart+ 倾向持续优化） |

## 七、CBO vs 广告组预算

### 7.1 概念对比

| **CBO (Campaign Budget Optimization)** | **Ad Group Budget（广告组预算 / ABO）** |
|---|---|
| 在 Campaign 层级设置一个总预算，系统**自动**在多个 Ad Group 间动态分配 | 在每个 Ad Group 单独设置预算，人工控制每个组的花费 |
| 适合：Campaign 下 Ad Group 目的一致（都是跑转化） | 适合：每个 Ad Group 有独立预算控制需求 |
| 系统会把预算倾斜到表现最好的 Ad Group | 需要人工监控和调整 |
| **推荐默认选项**（TikTok 官方推荐） | 精细测试时使用 |

### 7.2 CBO 的优势

- 系统自动发现最优组合
- 省去人工调整 5+ 个 Ad Group 预算的时间
- 学习效率更高（总预算支持算法更快收敛）
- 与 Smart+ 天然契合

### 7.3 什么时候用 Ad Group Budget

- 需要严格控制某个定向/创意的测试预算（比如只给新创意 $20/天）
- 每个 Ad Group 的目标 CPA 不同（需要独立的成本目标）
- 对广告组间的预算分配有特殊策略要求（如强制等量分配做 A/B 测试）

## 八、Split Test（A/B 测试）

### 8.1 入口与支持的测试类型

- 入口：**Campaigns → Create → Create Split Test**（或顶部 Split Test 按钮）
- 系统支持的测试维度：
  - **创意测试 (Creative)** — 同一定向/出价下测试不同视频/文案
  - **受众测试 (Audience)** — 同一创意/出价下测试不同兴趣/行为标签
  - **出价策略测试 (Bidding Strategy)** — Lowest Cost vs Cost Cap vs Bid Cap
  - **版位测试 (Placement)** — Automatic vs Manual Placement
  - **预算测试 (Budget)** — 不同日预算规模

### 8.2 正确的 A/B 测试流程（5 步）

```
Step 1：明确假设（Hypothesis）
  → 例："视频 A 的 CTR 会比视频 B 高 20%，因为开头钩子更吸引人"

Step 2：设计测试结构
  → 同一 Campaign，两个 Ad Group（除测试变量外其余保持一致）
  → 预算：每组 ≥ $50/天，保证统计显著性
  → 时间：最短 7 天（给系统学习期 + 稳定期）

Step 3：运行测试
  → 同时上线，避免时间偏差
  → 中途不要改动非测试变量（会污染数据）

Step 4：判断结果
  → 统计显著性 ≥ 90%（p-value < 0.1）
  → Power ≥ 80%（样本量足够发现差异）
  → 关注 CPA / CTR / CVR 等核心指标

Step 5：应用结论
  → 胜者保留 + 扩预算
  → 失败者关停或修改后重新测试
```

### 8.3 常见错误

| ❌ 错误做法 | ✅ 正确做法 |
|------------|------------|
| 测试多个变量同时跑（Creative + Audience 一起改） | 单变量测试，每次只改一个维度 |
| 测试 1-2 天就下结论 | 最少 7 天，等 Learning Phase 结束 |
| 样本量不足判断胜负（< 50 次转化） | 使用统计显著性计算器，不要凭直觉 |
| 把 CTR 当作转化效果唯一指标 | 综合看 CPA、ROAS、CVR |
| 测试获胜创意后不替换现有素材 | 定期更新创意池，防止创意疲劳 |

## 九、学习期（Learning Phase）

### 9.1 什么是学习期

TikTok 系统给一个新 Ad Group 贴的标签。期间系统在**大规模探索用户特征**，因此 CPA 会不稳定，可能偏高。

**退出学习期的条件**（满足任一）：
- 7 天内累计 ≥ 50 次转化事件
- 或系统判定已收集足够数据（较少见）

### 9.2 学习期期间的行为建议

| ✅ 可以做 | ❌ 不要做 |
|----------|----------|
| 监控基础指标（Spend、Impressions、CTR） | 频繁修改预算（< 48 小时就调整） |
| 观察创意表现（完播率、3 秒跳出率） | 修改受众定向 |
| 补充素材（如果创意表现差） | 修改出价策略或竞价上限 |
| — | 暂停/重启 Ad Group（每次重启会重置学习期） |

> **72 小时规则**：新 Ad Group 上线后**前 72 小时**是最关键的学习窗口。此期间如非必要不要做任何设置修改。

### 9.3 学习失败的典型信号

- 7 天后 CPA 仍远高于行业均值
- 学习期标签持续存在（≥ 14 天未退出）
- 没有花费 / 花费远低于预算
- CTR < 1% 或转化率 < 0.5%

### 9.4 学习失败的排查思路

1. **检查受众**：定向是否过窄（< 100 万）？是否太宽（> 1 亿）但没有学习信号？
2. **检查预算**：日预算 < 目标 CPA × 20，样本量不足
3. **检查优化事件**：是否太靠后（如直接优化 Complete Payment 但每周 < 20 次）
4. **检查创意**：点击率过低导致系统没有有效信号
5. **检查 Pixel**：事件上报是否正常（去 Events Manager 看诊断报告）

## 十、在 TTAM 界面中的位置

- **出价策略选择器**：位于 Ad Group 创建/编辑页面的「预算与出价」区块
- **优化事件**：同一区块，出价策略下方
- **预算类型切换**（日预算/总预算）：出价上方
- **CBO 开关**：Campaign 创建页面顶部
- **Split Test 创建入口**：Campaigns 页面顶部 Split Test 按钮，或 Create 流程中选择 Create Split Test
